3.  **前端页面**：

# Goal
请在**不修改项目现有核心逻辑**的前提下，实现“热门景点”页面的完整功能。
功能点包括：
1.  **热门算法**：从数据库筛选出最热门的 **9个** 景点。
2.  **前端展示**：在 `hot.html` 动态渲染这 9 个景点卡片。
3.  **交互跳转**：点击卡片跳转详情页，增加返回首页按钮。

# Implementation Plan (请严格按此步骤生成代码)

## Task 1: Backend Implementation (Java & SQL)
请修改后端代码以提供热门景点数据。

1.  **Mapper 层 (`AttractionMapper.xml`)**:
    * 新增一个查询 `<select id="selectHotAttractions">`。
    * **排序算法 (Weighted Sort)**：请使用 SQL 的 `ORDER BY` 语句计算热度权重。
    * *推荐公式*：`ORDER BY (view_count * 0.7 + rating * 50 * 0.3) DESC`。
    * (说明：rating 满分5.0，乘以50变为250分制，避免浏览量(view_count)过大导致评分权重被忽略)。
    * 限制返回前 **9** 条数据 (`LIMIT 9`)。
2.  **Service 层 (`AttractionService`)**:
    * 新增方法 `List<Attraction> getHotAttractions();` 并实现它。
3.  **Controller 层 (`AttractionController`)**:
    * 新增接口 `GET /api/attraction/hot`。
    * 返回 `Result<List<Attraction>>`。

## Task 2: Frontend Implementation (`hot.html`)
请重写或完善 `hot.html` 的代码（HTML + CSS + JS）。

1.  **布局结构**：
    * **导航栏/头部**：包含一个醒目的“返回首页”按钮（链接到 `index.html`），可以使用简单的绝对定位或者 Flex 布局放在左上角。
    * **内容区**：使用 Grid 布局（3列 x 3行）或 Flex Wrap 布局，美观地展示 9 个景点卡片。
2.  **卡片样式**：
    * 每个卡片应包含：景点封面图 (`image_url`)、景点名称 (`name`)、评分 (`rating`)、浏览量 (`view_count`)。
    * 图片需设置统一宽高（`object-fit: cover`）以防止变形。
    * 卡片增加鼠标悬停效果（轻微放大或阴影加深）。
3.  **JS 逻辑 (jQuery)**：
    * 页面加载时 (`$(document).ready`) 自动调用 `/api/attraction/hot`。
    * 遍历返回的数据，生成 HTML 字符串并插入容器。
    * **点击事件**：为每个卡片绑定点击事件，跳转至 `detail.html?id={id}`。
